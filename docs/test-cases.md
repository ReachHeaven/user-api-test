# Тест-кейсы: User API (AdonisJS)

## POST /user/create

| ID | Заголовок | Предусловия | Тестовые данные | Шаги | Ожидаемый результат | Приор. | Сер. |
|----|-----------|-------------|-----------------|------|---------------------|:------:|:----:|
| TC-01 | Создание с валидными данными | стенд доступен | уникальные `username`/`email`, `password=Secret123` | POST FormData | `201`, тело содержит созданного пользователя с числовым `id`; запись сохраняется и видна в `GET /user/get` | P0 | Crit |
| TC-02 | Пароль не возвращается в ответе создания | — | валидные данные | POST, проверить тело | поля `password` нет | P0 | Crit |
| TC-03 | Схема тела ответа создания | — | валидные данные | POST, разобрать JSON | `success=true`, `details` с `id/username/email/created_at/updated_at`, корректное сообщение | P2 | Low |
| TC-04 | `username` отсутствует | — | без поля `username` | POST | `400`, ошибка обязательности | P1 | High |
| TC-05 | `username` пустой `""` | — | `username=""` | POST | `400` | P1 | High |
| TC-06 | `username` только пробелы | — | `username="   "` | POST | `400` (после trim — пусто) | P2 | Med |
| TC-07 | `username` дубликат | пользователь N создан | `username = N.username` | POST | `400`, «username занят» | P1 | High |
| TC-08 | `email` отсутствует | — | без поля `email` | POST | `400`, ошибка обязательности | P1 | High |
| TC-09 | `email` некорректного формата | — | `email="not-an-email"` | POST | `400` (нарушение формата) | P1 | High |
| TC-10 | `email` дубликат | пользователь N создан | `email = N.email`, другой `username` | POST | `400`, «email уже существует» | P1 | High |
| TC-11 | `password` отсутствует | — | без поля `password` | POST | `400`, ошибка обязательности | P1 | High |
| TC-12 | `password` короче минимума | — | `password="x"` | POST | `400` (нарушение парольной политики) | P1 | High |
| TC-13 | Все обязательные поля отсутствуют | — | пустой POST (без полей) | POST | `400` со **списком всех** ошибок | P2 | Med |
| TC-14 | Лишнее неизвестное поле | — | + `role=admin` | POST | поле игнорируется ИЛИ `400`; внутренности не раскрываются | P1 | High |
| TC-15 | Content-Type JSON вместо multipart | — | JSON-тело | POST `application/json` | по контракту FormData: `400/415` или осознанная поддержка | P2 | Med |
| TC-16 | Строки-инъекции как данные | — | `username` c `' OR 1=1`, `<script>` | POST | значение сохраняется как данные, без `500`/выполнения | P2 | Med |
| TC-17 | Превышение максимальной длины полей | — | `username`/`email` длиной 10 000+ | POST | `400` при превышении лимита | P3 | Low |
| TC-18 | Неверный HTTP-метод на `/user/create` | — | GET на `/user/create` | GET | `404/405` | P3 | Low |

## GET /user/get

| ID | Заголовок | Предусловия | Тестовые данные | Шаги | Ожидаемый результат | Приор. | Сер. |
|----|-----------|-------------|-----------------|------|---------------------|:------:|:----:|
| TC-19 | Список отражает созданного пользователя | создан пользователь N | — | POST N → GET | в массиве есть N с его `username`/`email` | P0 | Crit |
| TC-20 | Контракт ответа: статус и тип | — | — | GET, читать заголовки | `200`, `application/json`, тело — JSON-массив | P0 | High |

| TC-22 | Схема элемента списка | создан ≥1 пользователь | — | GET, проверить поля | `id, username, email, created_at, updated_at` | P1 | Med |
| TC-23 | POST допускается на `/user/get` | — | POST на `/user/get` | POST | `404/405` (маршрут только GET) | P2 | Med |
| TC-24 | Время ответа GET | — | — | GET, замер | ответ в пределах SLA (напр. < 2с) | P3 | Low |
